# Module 6 — Momentum Mode

## Purpose

Reduce task initiation resistance — the ADHD experience of knowing what needs to be done but being unable to start.

## The Core Shift

Most productivity tools ask:

> **"What do you want to do?"**

This question requires the user to make a decision under the weight of their full backlog — a recipe for paralysis.

Momentum Mode asks instead:

> **"What is the smallest next step?"**

This reframes the task from a large, overwhelming commitment into a single, concrete, low-stakes action.

## Workflow

1. User signals they are stuck or can't start
2. System surfaces the current priority
3. System asks: *"What is the smallest possible first action on this?"*
4. User defines a micro-step (e.g., "open the document")
5. System confirms and starts a session
6. Momentum builds from there

## Design Rationale

ADHD task paralysis is often caused by the brain perceiving a task as too large or ambiguous to begin. Breaking it into the smallest possible unit bypasses this defence mechanism. Momentum Mode is not a planning tool — it is a starting tool.
