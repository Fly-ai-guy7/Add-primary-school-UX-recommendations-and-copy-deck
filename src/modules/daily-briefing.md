# Module 3 — Daily Briefing

## Purpose

Provide structured orientation at the start and end of each day, replacing the overwhelming blank-slate problem with a clear, curated view of what matters.

---

## Morning Briefing

Delivered at a user-configured time each morning.

**Displays:**

| Section | Content |
|---|---|
| **Top 3 priorities** | AI-selected highest-impact tasks for the day |
| **Calendar events** | Scheduled meetings and appointments |
| **Pending follow-ups** | Items awaiting action from previous days |
| **Estimated workload** | Projected time demand for the day's tasks |

---

## Evening Briefing

Delivered at end-of-day or session close.

**Displays:**

| Section | Content |
|---|---|
| **Completed items** | Everything finished today |
| **Carry-forward items** | Unfinished tasks moved to tomorrow |
| **Daily progress score** | Momentum metric reflecting execution quality |

---

## Design Rationale

Time blindness is a core ADHD challenge. Daily briefings create external temporal anchors — concrete start-of-day and end-of-day rituals that orient the user and prevent days from blurring together without progress.
