# UX Recommendations — Overview

These recommendations govern the design and interaction patterns across all FocusFlow AI™ surfaces. Every decision is grounded in the cognitive profile of ADHD users.

## Guiding Principles

### 1. Reduce Friction Before It Appears

ADHD users abandon tools the moment they encounter resistance. Every extra tap, every required field, every loading spinner is a potential exit point. Design to eliminate friction proactively, not reactively.

### 2. Never Ask Users to Organise

Organisation is the skill ADHD impairs. FocusFlow must accept raw, unstructured input and return structure. The system does the sorting, labelling, and prioritising — not the user.

### 3. One Thing at a Time

The interface should never compete for attention with itself. Show one action, one task, one prompt at a time. Additional information should be accessible but never intrusive.

### 4. Calm Over Urgency

Avoid red alerts, urgent badges, and countdown timers. These trigger anxiety in ADHD users and are counterproductive. Use calm, neutral visual language even for important information.

### 5. Progress Over Perfection

Celebrate small wins. Every completed task, every brain dump processed, every session started deserves acknowledgement. Momentum is the product.

### 6. Forgiveness by Default

Nothing the user does should create irreversible negative consequences. Mistakes are undoable. Dismissed tasks reappear. Skipped sessions don't penalise. The system is patient.
