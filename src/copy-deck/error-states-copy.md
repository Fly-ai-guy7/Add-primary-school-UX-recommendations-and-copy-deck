# Copy Deck — Error States

**Rule:** Errors are never the user's fault in copy. Language is always solution-forward. Never use the word "error" in user-facing text.

---

## Network and Connectivity

| Scenario | Headline | Body | CTA |
|---|---|---|---|
| No internet connection | You're offline. | Your data is saved. We'll sync when you're back. | Got it |
| Slow connection | Taking longer than usual. | Hang on a moment. | *(none — auto-dismiss)* |
| Sync failed | Couldn't sync right now. | Your work is saved on this device. | Try again |

## Generic Fallback

| Element | Copy |
|---|---|
| Headline | Something went wrong. |
| Body | We're looking into it. Try again in a moment. |
| CTA | Try again |
| Support link | Contact support |

---

## Tone Note

Error copy uses sentence case, never all-caps. Never bold the bad news — bold the solution. Avoid the passive voice where possible ("We couldn't save that" rather than "That couldn't be saved").
